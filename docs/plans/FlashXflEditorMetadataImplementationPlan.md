# Flash XFL Editor Metadata Implementation Plan

## Status

Completed on March 6, 2026.

## Phase 1: Model expansion

- Add layer outline metadata to `TimelineLayer`.
- Add cache-as-bitmap metadata to `TimelineLayer`.
- Add richer text-field metadata to `LayerTextSettings`.
- Add any supporting enums required for text anti-alias / render modes.

## Phase 2: Sampling and rendering

- Carry the new layer display metadata through `LayerSnapshot`.
- Render password fields safely in stage/export previews.
- Respect max-character limits in preview.
- Respect outline mode in stage/export previews.

## Phase 3: XFL import/export

- Export/import layer outline metadata on `DOMLayer`.
- Export/import symbol cache metadata on symbol instances.
- Export/import advanced text-field attributes on text elements.
- Keep import backward-compatible with older XFL generated by previous app versions.

## Phase 4: Editor wiring

- Add inspector controls for layer outline and cache metadata.
- Add text-field controls for max chars, password, device fonts, and anti-alias mode.
- Surface concise summaries in the library/selection UI where useful.

## Phase 5: Validation

- Add regression tests for:
  - outline metadata round-trip
  - instance cache metadata round-trip
  - text-field metadata round-trip
  - password/max-char preview behavior
- Run build, tests, and a desktop smoke launch.

## Acceptance criteria

- XFL round-trip preserves layer outline mode/color.
- XFL round-trip preserves cache-as-bitmap metadata on instances.
- XFL round-trip preserves text max-char, password, device-font, and anti-alias metadata.
- The editor inspector can view and edit all new fields.
- Stage/export preview reflects the new text and outline behaviors in a deterministic way.
